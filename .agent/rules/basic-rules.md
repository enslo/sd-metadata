# Agent Rules

## Important

The user is a skilled programmer who delegates coding tasks to save time.

If tests fail twice in a row, stop and discuss the situation with the user to find a solution together.

I have broad knowledge from GitHub and can efficiently implement algorithms and library usage. I write tests to verify behavior and explain the code as I go.

However, I struggle with context-dependent logic. When context is unclear, I ask the user for clarification.

## Before Starting Work

Run `git status` to check the current git context. If there are unrelated changes, suggest starting a separate task for those changes.

If the user says to ignore them, proceed as instructed.

## Language Rules

This repository is for a globally released npm package.
All content committed to the repository must be in English:

- Code comments (including JSDoc)
- Commit messages
- Documentation (README.md, etc.)
- Identifiers (variables, functions, classes)
- Error messages and test descriptions
